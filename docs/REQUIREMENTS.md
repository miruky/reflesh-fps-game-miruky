# hibana 要件定義書

制定日: 2026-06-11

hibanaはブラウザで動作する3D FPSである。本書は発注時の全要求機能を漏れなく記録し、Web環境への適合方針と実装フェーズを定める。機能の取捨選択を行う場合は本書を更新してから行う。

## 技術選定

- 描画: Three.js(WebGL)。Unreal Engine 5はHTML5書き出しを公式サポートしておらず、無料Web配信の要件を満たせないため不採用とした。
- 物理: Rapier(Rust製エンジンのWASMビルド)。キャラクターコントローラとレイキャストに使用。
- 言語: TypeScript(strict)。
- 配信: GitHub Pages(静的サイトとして完全無料)。代替としてNetlifyにも同一成果物を配置できる。
- マルチプレイヤー: WebRTC DataChannelによるP2P+ホスト権威方式を採用予定。GitHub Pagesはサーバープロセスを持てないため、専用サーバー方式は採らない。シグナリングは無料枠で動く軽量サービスを別リポジトリとして用意する。
- サウンド: Web Audio APIによる全面プロシージャル生成。音声アセットを持たない。

## ステージ要件

10ステージを実装する。シード付き決定論的ジェネレータでレイアウトを生成し、ステージごとに固有のパレット(空・霧・床・壁・障害物・アクセント色)、広さ、障害物密度、高低差を与える。点対称配置により対戦時の公平性を担保する。

## 機能要件(全項目)

### 1. プレイヤー操作・移動系

- 基本移動: 前後左右移動(WASD)、視点操作(マウス/右スティック)
- ジャンプ / しゃがみ / 伏せ(プローン)
- スプリント(ダッシュ)と移動速度の状態管理(歩き・走り・ADS中の減速)
- スライディング / バニーホップ / 慣性制御
- よじ登り(マントリング)/ 乗り越え(ヴォールト)
- リーン(覗き込み)
- 梯子・ロープ・ジップライン等の特殊移動
- 泳ぎ / 落下ダメージ判定
- 足音・着地音の状態連動(しゃがみ歩きで静かになる等)
- コヨーテタイム的な入力猶予、入力バッファ

### 2. 射撃・武器系

- 発射処理: ヒットスキャン / 弾道シミュレーション(プロジェクタイル、重力落下、弾速)
- ADS(エイムダウンサイト)/ 腰だめ撃ちの切替と精度差
- リコイル(反動)パターンとスプレッド(拡散)
- リロード(タクティカルリロード / 空リロード、リロードキャンセル)
- 弾薬管理: マガジン残弾、所持弾数、弾薬タイプ
- 発射モード切替: 単発 / バースト / フルオート
- 武器切替: プライマリ / セカンダリ / 近接、切替速度
- 武器ごとのパラメータ: ダメージ、レート、射程、ダメージ減衰、貫通
- ヒット判定: ヘッドショット倍率、部位ダメージ、壁貫通
- ヒットフィードバック: ヒットマーカー、ダメージ数値、キル確認音
- 近接攻撃
- 投擲物: グレネード(フラグ / スモーク / フラッシュ / 焼夷)、調理(クッキング)、投擲軌道表示
- 武器アタッチメント / カスタマイズ: サイト、サプレッサー、グリップ等
- エイムアシスト(コントローラー向け: 吸い付き、減速)
- クロスヘア: 動的拡縮、カスタマイズ

### 3. キャラクター状態・戦闘ルール

- HP / シールド / アーマーとダメージ計算
- 回復システム: 自動回復、回復アイテム
- ダウン(瀕死)状態と蘇生(チーム戦)
- 死亡 / リスポーン処理: リスポーン地点、リスポーンウェーブ
- 状態異常: スタン、フラッシュ(白飛び)
- 被弾方向インジケータ
- スキル / アビリティ / アルティメット
- クラス / ロードアウトシステム
- キルストリーク / スコアストリーク報酬

### 4. カメラ・視点

- 一人称カメラ: FOV設定、頭部ボブ、カメラシェイク
- 一人称用ビューモデル(腕+武器の専用モデル、FOV分離)
- 三人称への切替
- 観戦(スペクテイター)カメラ: 死亡後観戦、フリーカメラ、視点切替
- キルカメラ / リプレイ
- ADS時のズーム、スコープ表現(ピクチャーインピクチャー含む)

### 5. UI / HUD

- HUD: HP/シールド、残弾、武器スロット、ミニマップ/コンパス、スキルCT、スコア、残り時間
- キルログ(キルフィード)
- ダメージ表示・ヒットマーカー
- 目標マーカー / ウェイポイント / ピン機能
- スコアボード / 戦績画面(TAB)
- メニュー: メイン、ポーズ、設定、ロードアウト編成
- マップ全体表示
- リザルト画面: MVP、戦績、獲得XP
- デスリキャップ(死因表示)

### 6. マルチプレイヤー / ネットワーク

- クライアント・サーバーアーキテクチャ(専用サーバー / P2P)
- ラグ補償: クライアント予測、サーバーリコンシリエーション、補間、巻き戻し判定
- ティックレート管理
- マッチメイキング: スキルベース(SBMM)、パーティ、リージョン選択
- ロビー / ルーム機能: ルーム作成、パスワード、カスタムマッチ
- ホストマイグレーション(P2P系)
- チート対策(アンチチート): サーバー権威、異常検出、リプレイ検証
- 通信系: ボイスチャット(VC)、テキストチャット、ミュート/ブロック、通報機能
- ネット状態表示: Ping、パケットロス、接続品質警告
- 再接続(リコネクト)機能
- 観戦・配信モード(ストリーマーモード: 名前隠し等)

### 7. ゲームモード・ルール

- 代表的モード: チームデスマッチ、フリーフォーオール、ドミネーション(拠点制圧)、爆破(サーチ&デストロイ)、ペイロード護衛、CTF、バトルロイヤル、ガンゲーム
- ラウンド管理: ラウンド制、サイドチェンジ、オーバータイム
- 勝利条件・スコア計算
- チーム分け / オートバランス
- キャンペーン / ソロモード: チェックポイント、難易度選択
- Co-opモード(ゾンビ、ホード等)
- 練習場 / 射撃訓練場 / BOTマッチ / チュートリアル

### 8. AI(シングル / BOT)

- 敵AI: 索敵(視覚・聴覚)、カバー利用、難易度調整
- 経路探索(ナビメッシュ)
- BOT補充(マルチの人数合わせ)
- 味方AI / コンパニオン(キャンペーン用)
- AIディレクター的な動的難易度

### 9. レベル・環境

- スポーンシステム: スポーン地点ロジック(敵との距離、視線考慮)
- インタラクト: ドア、スイッチ、アイテム取得、設置(爆弾、リスポーンビーコン)
- 破壊可能オブジェクト / 環境破壊
- アイテム / 武器のドロップ・ピックアップ
- 危険地帯: 落下死エリア、毒ガス、収縮サークル
- オクルージョンカリング、LOD等の最適化
- マップローテーション / 投票

### 10. アニメーション・演出

- 一人称アニメ: 発射、リロード、武器切替、検品(インスペクト)、走りモーション
- 三人称アニメ(他プレイヤーから見えるモーション)、移動ブレンド
- ラグドール / 死亡演出
- VFX: マズルフラッシュ、弾痕(デカール)、薬莢排出、トレーサー、血しぶき/ヒットエフェクト、爆発
- 被弾演出: 画面の血のり、ビネット、低HP時の演出

### 11. サウンド

- 3D空間オーディオ / 立体音響(足音の方向・距離)
- 銃声: 距離減衰、屋内外での残響差、サプレッサー音
- 足音: 材質別(金属・木・草)、自他の区別
- オクルージョン(壁越しのこもり)
- アナウンス / VO: キル通知、目標状況、キャラボイス
- BGMの動的制御
- 音量カテゴリ別設定(VC、SE、BGM個別)

### 12. 進行・メタゲーム

- アカウント / プロフィール / レベル・XP
- アンロック: 武器、アタッチメント、スキン
- バトルパス / チャレンジ / デイリーミッション
- ランクシステム(レーティング)とシーズン制
- 統計(スタッツ): K/D、勝率、命中率、ヒートマップ
- コスメティック: スキン、エンブレム
- ストア / 通貨管理
- フレンド / パーティ / クラン機能
- リプレイ保存・theaterモード

### 13. 設定・アクセシビリティ

- 入力設定: キーバインド変更、マウス感度、コントローラーのデッドゾーン/レスポンスカーブ、ジャイロ対応
- グラフィック設定: 解像度、フレームレート上限、垂直同期、シェーダー品質、モーションブラーOFF
- アクセシビリティ: 色覚サポート(敵ハイライト色変更)、字幕、UIスケール、ホールド/トグル切替、画面揺れ軽減
- 言語ローカライズ
- クロスプレイ / クロスプログレッション
- 入力デバイス別マッチング

### 14. 基盤・運用系

- セーブ / クラウドセーブ
- パッチ配信・バランス調整の仕組み(武器パラメータの外部データ化)
- テレメトリ / 分析基盤
- サーバー負荷分散・スケーリング
- 不具合通報・ペナルティシステム(AFK検出、トロール対策、チームキル処理)
- 年齢レーティング対応(ゴア表現のON/OFF等)

## Web環境への適合方針

完全無料・ブラウザ動作という制約に対し、原義どおりの実装が不可能な項目は以下のとおり読み替える。読み替えはここに列挙したものに限る。

- カーネルレベルアンチチート: ブラウザでは実装不能。ホスト権威の判定、入力統計の異常検出、リプレイ検証で代替する。
- ストア / 課金(IAP): 実通貨の課金は行わない。ゲーム内通貨とアンロクのみで構成する。
- 専用サーバー / サーバースケーリング: 常駐サーバーを持たないため、P2Pホスト権威+ホストマイグレーションで代替する。テレメトリはローカル保存を基本とする。
- クラウドセーブ: localStorageへの保存+エクスポート/インポート(JSONファイル)で代替する。
- コントローラー対応: Gamepad APIで実装する(ジャイロはブラウザAPIの制約内で対応)。

## 実装フェーズ

- P0 コア戦闘ループ: 移動・射撃・BOT戦・ステージ10種・HUD・メイン/ポーズメニュー・設定・プロシージャルサウンド
- P1 戦闘拡張: 投擲物、近接、武器追加、アタッチメント、スライディング、マントリング、リーン、部位ダメージ詳細、壁貫通
- P2 モード拡張: TDM/FFA/ドミネーション等のルール実装、リスポーンロジック、リザルト、観戦
- P3 ネットワーク: WebRTC P2P対戦、ラグ補償、ロビー、チャット、VC
- P4 メタゲーム: XP・アンロック・チャレンジ・統計・ランク
- P5 仕上げ: アクセシビリティ全項目、ローカライズ、リプレイ、キルカメラ
